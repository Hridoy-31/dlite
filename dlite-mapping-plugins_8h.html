Detailed Description

Common API for all mapping plugins (internal).

A DPite mapping plugin should be a shared library that defines the function 

const DLiteMappingPlugin *
get_dlite_mapping_api(const char *name);

that returns a pointer to a struct with pointers to all functions provided by the plugin. The name is just a hint that plugins are free to ignore. It is used by mapping plugins that supports several different drivers, to select which api that should be returned.

The mapping plugin search path is initialised from the environment variable DLITE_MAPPING_PLUGIN_DIRS.

◆ dlite_mapping_plugin_get()

const DLiteMappingPlugin* dlite_mapping_plugin_get ( const char *  name)

Returns a mapping plugin with the given name, or NULL if it cannot be found.

If a plugin with the given name is registered, it is returned.

Otherwise the plugin search path is checked for shared libraries matching name.EXT where EXT is the extension for shared library on the current platform ("dll" on Windows and "so" on Unix/Linux). If a plugin with the provided name is found, it is loaded, registered and returned.

Otherwise the plugin search path is checked again, but this time for any shared library. If a plugin with the provided name is found, it is loaded, registered and returned.

Otherwise NULL is returned.
